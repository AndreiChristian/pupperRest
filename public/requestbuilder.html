Request Builder

For our API, chaining allows users to apply multiple filters to their requests to narrow down the returned results to meet their specific criteria. When forming the request URL, the query string is structured with distinct key-value pairs, each representing a separate filter. These pairs are separated by an ampersand (&), forming a chain of conditions. Each key in the pair represents a particular attribute of the data (like 'size', 'origin', or 'trainability'), and the value is what you're specifically looking for in that attribute. For example, if you wanted to find dogs that are of 'small' size and are 'high' in trainability, you'd construct a query string like: `?size=small&trainability=high`. When the API receives this request, it first filters the data based on the 'size' attribute, then takes that subset of data and filters it again based on 'trainability'. The result will be a collection of entries that meet both conditions - they are 'small' and have 'high' trainability. It's crucial to understand that the order of the filters does not matter; the result will always be the same. Also, while the chaining mechanism allows you to specify multiple filters, you must always ensure that the filters are relevant to the data you are requesting. If a filter is irrelevant, it will not break the request but may return an empty result if no entries meet the specified criteria.

Examples

#1

https://expressjs-postgres-production-f8f8.up.railway.app/api/filter/?origin=United Kingdom

In this example, the request is searching for dog breeds that originated in the United Kingdom. To do this, it adds the "origin" key to the query string, with the value being "United Kingdom". Therefore, the generated URL only returns breeds that fit this specific criteria.

#2

https://expressjs-postgres-production-f8f8.up.railway.app/api/filter/?size=Medium&origin=United Kingdom&trainability=Low

In this example, the request is further refined to search for medium-sized dog breeds that originated in the United Kingdom and have a low trainability. The attributes "size", "origin", and "trainability" are included in the query string. Each of these attributes is assigned a specific value (Medium, United Kingdom, and Low respectively), effectively narrowing down the search results to only include breeds that meet all these criteria.

#3

https://expressjs-postgres-production-f8f8.up.railway.app/api/filter/?purpose=Flushing/Retrieving&coat_type=Short-haired&origin=Sweden&temperament=Smart

In this example, the request is searching for dog breeds with specific characteristics: a purpose of flushing/retrieving, a short-haired coat type, origin from Sweden, and a smart temperament. To accomplish this, the "purpose", "coat_type", "origin", and "temperament" keys are included in the query string. These keys are each assigned a specific value (Flushing/Retrieving, Short-haired, Sweden, and Smart respectively). Therefore, the generated URL only returns breeds that fit all these specific criteria.
