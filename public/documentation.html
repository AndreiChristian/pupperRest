Pupper Rest API Documentation

Overview

The Pupper Rest API is a non-commercial project focused on delivering comprehensive data on a variety of dog breeds. This API is perfect for those who share our love for canines and wish to access detailed information about different breeds.

Getting Started

If you are unfamiliar with making API requests, don't worry. Our user-friendly Request Builder will guide you through the process, facilitating the creation and execution of requests.

API Endpoints

Our API comprises several endpoints that provide access to our extensive dog data.

The base URL is https://expressjs-postgres-production-f8f8.up.railway.app/api/

https://expressjs-postgres-production-f8f8.up.railway.app/api/all

The /all endpoint retrieves all available data for every dog breed in our database. Send a GET request to this endpoint to receive a JSON object containing breed IDs and names.

https://expressjs-postgres-production-f8f8.up.railway.app/api/filter/?...options

The /filter endpoint allows you to customize your queries based on specific parameters. For instance, you might wish to filter breeds based on size, use, or purpose. To do so, simply send a GET request to this endpoint and include your chosen parameters in the request URL. Please consult the Request Builder for detailed instructions on using this feature.

https://expressjs-postgres-production-f8f8.up.railway.app/api/:id

If you need information about a particular breed, use the /:id endpoint. Replace :id with the desired breed's ID and send a GET request to receive a comprehensive JSON object with the breed's details.

Data Format

In line with our commitment to simplicity, all data returned by our API is in JSON format. This choice of data format ensures compatibility with most programming languages and ease of use.

Rate Limiting

To ensure fair access to all users, we have instituted a rate limit of 2 requests per second and 50 requests per hour per user. Overstepping these limits may result in temporary access suspension. In such cases, users will see a "Too many requests, please try again later." message.

Caching

To enhance the performance of your applications and minimize server load, we recommend implementing caching. This strategy speeds up data retrieval and reduces server strain, resulting in a more efficient overall system.

Feedback and Contributions
