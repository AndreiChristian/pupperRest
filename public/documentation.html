Hey there, welcome to the Pupper Rest API Docs!

So, what's this all about?

Here's the deal - we love dogs. So, we decided to make the Pupper Rest API, a fun project all about sharing data on a wide variety of dog breeds. We made it for non-commercial use, so it's really all about sharing the canine love.

How do I dive in?

Not sure how to start making requests? No worries at all! Our nifty Request Builder is here to help. It's a super easy tool that'll guide you through making requests.

API Endpoints

We've set up a few different endpoints for you to access all our awesome dog data.

The base URL is https://expressjs-postgres-production-f8f8.up.railway.app/

1. /all

This is the go-to endpoint to fetch all the data about every single dog breed we have. Just send a GET request to /all and you'll get back a JSON object full of canine goodness.

2. /filter

Now, we understand that sometimes you're looking for something specific. That's where the /filter endpoint comes in. Send a GET request to /filter along with your parameters (like size, use, or purpose) and we'll return a tailored list of dog breeds that match your filters ( visit Request Builder for more info).

3. /:id

Want info about a specific dog breed? We got you covered. Just replace :id with the ID of the dog breed you're interested in and send a GET request to /:id. You'll receive a JSON object containing all the data we have about that breed.

What's in the box?

We're all about simplicity. That's why our API only sends back data in JSON format. It's easy to use, flexible, and works well with most programming languages. So, you can focus on the fun part – playing with the data!

Share the love

We've got loads of data but need to make sure all users get a fair go. That's why we've set a limit of 2 requests per second and 50 requests per hour for each user. Please stick to these limits to keep things fair. If you go over, we'll have to temporarily block your access and you'll see a "Too many requests, please try again later." message. It's all about playing fair, folks!

Efficiency for the win!

To help your apps run faster (and keep our server from getting overwhelmed), it'd be awesome if you could implement some caching. It reduces the load on our server and helps you get the data faster. It's a win-win!

Like what we do?

If you're finding our API useful and want to show some appreciation, a simple mention goes a long way for us. And if you want to go the extra mile and contribute, we'd be over the moon! Feel free to make a pull request – we're always open to improvements.
